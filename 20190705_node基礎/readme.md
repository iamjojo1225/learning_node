### 2.數據交互

	* #### HPPT-協議

	* HPPT 1.0 RFC-1945
	* HPPT 1.1 RFC-2616	持久連接
	* HPPTS	 RFC-2888	安全協議
	* HPPT 2.0 RFC-7450	加密、頭部壓縮、服務器推送、管線操作、多路實用

#### HPPT封包結構

	##### 結構
	herder
	訊息
	<=32K

	body
	數據
	<=2G

狀態碼:

	* 1xx 訊息(websocket會用到)
	* 2xx 成功
	* 3xx 重定向(轉址)
	* 4xx 請求錯誤
	* 5xx 伺服器錯誤

請求方法:

	GET	獲取數據
		數據放在url內傳輸
		1.容量小<=32k

	POST	發送數據
		1.容量大
